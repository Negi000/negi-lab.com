    営業日計算は土日と内蔵の日本の祝日を除外します。重要な締切や法的・医療・金融上の判断には、必ず公式情報や専門家の確認を併用してください。

    よくある質問

    入力した日付は送信されますか？

    通常の計算はブラウザ内で完結します。計算履歴はこの端末のlocalStorageにのみ保存され、履歴削除ボタンで消せます。

    営業日計算はどこまで対応していますか？

    土日と内蔵の日本の祝日データを除外します。会社独自の休日や最新の祝日変更は、必要に応じて公式カレンダーで確認してください。

    マイナスの日数も使えますか？

    はい。日付の加算・減算では、-5 のように入力すると基準日より前の日付を計算できます。

    日付計算で確認したいポイント

    ブラウザ内の日付計算で迷いやすい、含める日付・営業日・祝日の扱いを整理します。

    日付計算の注意点

    うるう年の計算規則

    4年に1度、100年に1度は平年、400年に1度はうるう年という規則があります。

    例：2000年（うるう年）、1900年（平年）、2024年（うるう年）

    営業日・祝日の複雑性

    このツールは内蔵の日本の祝日データを使います。会社独自の休業日は別途確認してください。

    対応：ハッピーマンデー・春分の日・秋分の日・天皇誕生日変更

    日付境界

    このツールは日付単位で計算します。時刻や国際タイムゾーンを含む厳密な計算には向きません。

    注意：海外拠点や時刻を含む期限は、現地の基準で確認してください

    祝日考慮 国民の祝日・振替休日除外

    • • 開始日・終了日の扱い：期間に含むか否かの明確化
    • • 月末日の処理：月単位の契約では別途規定を確認
    • • 時刻の考慮：このツールは日付のみを扱う
    • • 重要な期限：公式情報や社内規定を併用する

    よくある質問・トラブルシューティング

    日付計算時によくある疑問や問題の解決方法をまとめました。

    営業日計算で休日が正しく反映されない

    休日データの更新と確認：

    国民の祝日の変動要因

    • • 天皇誕生日：2019年に12/23→2/23に変更
    • • 振替休日：日曜祝日などに伴う振替日の確認
    • • ハッピーマンデー：成人の日・海の日・敬老の日等
    • • 春分・秋分の日：年によって20日・21日・22日・23日に変動

    確認・対処方法

    • • 内閣府カレンダー：公式の祝日情報を定期確認
    • • 企業独自休日：年末年始・夏季休暇の反映
    • • 業界特有の休日：銀行休業日・証券取引所休場日
    • • 地域特有の休日：地方自治体の独自休日
    勤続期間や経過日数で1日ずれる

    日付の境界・基準日の扱い：

    基準日の扱い

    • • 開始日を含む：開始日当日を1日目として数える
    • • 開始日を含まない：翌日から数え始める
    • • 終了日の扱い：締切当日を含むか事前に確認する
    • • 時刻の注意：このツールは日付単位で確認する

    期間計算の起算日

    • • 勤続年数：入社日を含むか否かの社内規定確認
    • • 契約期間：開始日・終了日の解釈統一
    • • 休暇関連：社内規定や法令の確認を併用
    • • 試用期間：入社日から3ヶ月・6ヶ月の満了日計算
    プロジェクト期間の見積もりが曖昧になる

    実用的な期間見積もり手法：

    見積もりを安定させるコツ

    • • バッファ期間：計算期間の10-20%を予備期間に設定
    • • 段階的マイルストーン：月次・週次での進捗確認点
    • • リスク要因の明確化：祝日・年末年始の影響度評価
    • • 過去データ活用：類似プロジェクトの実績ベース調整

    計画立案のベストプラクティス

    • • 逆算型計画：締切から必要期間を逆算して開始日決定
    • • クリティカルパス：最重要タスクの期間確保を優先
    • • 並行作業の活用：依存関係のないタスクは同時進行
    • • レビュー期間確保：成果物チェック・修正期間を含める
    海外展開時のタイムゾーン・現地祝日対応

    国際対応の実装指針：

    多国籍プロジェクト管理

    • • 基準時刻の統一：UTC基準での日程管理・各国で現地時刻表示
    • • 現地祝日の考慮：国別・州別の祝日カレンダー整備
    • • 夏時間対応：3月・11月の時刻変更期間での計算注意
    • • 文化的配慮：宗教的祭日・ラマダン等の考慮

    推奨：グローバル展開時は現地法務・人事部門と連携し、 正確な休日情報・労働基準法の確認が不可欠です
